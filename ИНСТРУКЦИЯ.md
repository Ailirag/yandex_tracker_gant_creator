# Планировщик задач ONE — инструкция для новичка

Пошагово и без предположений о том, что вы уже что-то знаете. Делайте сверху вниз.

> **Что делает программа.** Берёт задачи из очереди ONE Яндекс Трекера,
> раскладывает их по календарю с учётом важности, доступного ресурса команд
> (СА и разработчики, с отпусками) и графика релизов, и считает для каждой
> задачи даты **начала**, **окончания** и **плановую дату завершения**.
> Сначала — только в отчёт (Excel). В Трекер ничего не пишется, пока вы явно
> не попросите.

---

## 0. Что нужно один раз (настройка)

На рабочем компьютере всё это уже настроено — раздел нужен для нового ПК.

1. **Установить `uv`** (менеджер запуска Python). Открыть PowerShell и выполнить:
   ```powershell
   winget install astral-sh.uv
   ```
   Проверить: `uv --version` — должна показаться версия. Python отдельно ставить
   не надо, `uv` сам скачает нужный.

2. **Указать доступ к Трекеру** — две переменные окружения (задаются один раз):
   - `YATRACKER_TOKEN_GT` — OAuth-токен Яндекс Трекера;
   - `YATRACKER_ORGID_GT` — ID организации.

   На рабочем ПК они уже заданы. На новом — задать через «Изменение системных
   переменных среды» → «Переменные среды» → создать обе на уровне пользователя,
   затем **перезапустить** PowerShell.

3. **Открыть папку программы** в PowerShell (одна команда на каждый сеанс):
   ```powershell
   cd "D:\Claude\yandex tracker analyzer\planner"
   ```

Всё. Дальше — обычная работа.

## 1. Получить план (безопасно, ничего не меняет)

Это режим **dry-run**: программа только читает Трекер и кладёт отчёт в папку
`out\`. В Трекере ничего не меняется.

```powershell
uv run planner plan --capacity file --capacity-file "..\source\ресурс по командам по дням.xlsx"
```

Что произойдёт:
- программа скачает задачи (1–2 минуты, это нормально);
- в конце напишет, сколько задач запланировано и **пути к файлам** в `out\`:
  - `plan_ДАТА.xlsx` — **отчёт-таблица** (см. шаг 2);
  - `gantt_ДАТА.html` — **диаграмма Ганта** по командам: откройте двойным
    кликом в браузере — полоски задач по фазам, релизы, дедлайны, линия
    «сегодня», детали по наведению мышью;
  - `plan_ДАТА.json` — то же для машины (нужно для записи в Трекер);
  - `backup_ДАТА.json` — снимок текущих дат (страховка).

> Без `--capacity file ...` программа посчитает по «прикидочной» численности
> (мок), без учёта отпусков. С файлом из 1С — по реальной доступности.
> Рекомендуется всегда указывать файл.

### Полезные варианты (по желанию)

```powershell
# план с конкретной даты (по умолчанию — сегодня)
uv run planner plan --capacity file --capacity-file "..\source\ресурс по командам по дням.xlsx" --start 2026-08-01

# убрать из плана «шум»: задачи в работе и на завершении
... --exclude-in-progress --exclude-tail

# только определённые задачи (язык запросов Трекера)
... --query '"Queue": "ONE" "Status": "Готово к разработке"'
```

## 2. Прочитать отчёт (Excel)

Откройте `out\plan_ДАТА.xlsx`. В нём 4 листа.

**Лист «План»** — главный. Одна строка = одна задача. Важные колонки:

| Колонка | Что значит |
|---|---|
| Ключ, Название | задача в Трекере |
| Команда, Системы | к какой команде отнесена и в какой системе релиз |
| Важность | чем **меньше число — тем важнее** (порядок выполнения) |
| СА старт / СА конец | когда аналитик занят задачей |
| Dev старт / Dev конец | когда разработчик занят задачей |
| Тест-буфер до | +5 рабочих дней на тестирование после разработки |
| Релиз (Р) | ближайшее релизное окно системы |
| **start: расчёт** | предлагаемая дата начала |
| **end: расчёт** | предлагаемая дата окончания |
| **ПДЗ: расчёт** | предлагаемая плановая дата завершения (привязана к релизу) |
| start/end/ПДЗ: **было** | что сейчас стоит в Трекере (для сравнения) |
| Предупреждения | если с задачей что-то не так — читайте здесь |

Колонки «расчёт» подсвечены жёлтым, если отличаются от «было».

**Лист «Пропущенные»** — задачи, которые НЕ попали в план, и **почему**
(нет важности, нет оценки, команда без ресурса и т.п.). Это ваш список
«что дозаполнить в Трекере».

**Лист «Смещения»** — если это не первый прогон: какие задачи сдвинулись
относительно прошлого раза и из-за кого.

**Лист «Инфо»** — предупреждения по всему прогону и источник данных.

На этом для «просто посмотреть план» всё. Файл можно переслать, распечатать,
обсудить. В Трекере по-прежнему ничего не изменилось.

## 3. «А что будет, если…» (проверка без риска)

Хотите понять, как изменится график, если задаче поднять важность:

```powershell
uv run planner plan --capacity file --capacity-file "..\source\ресурс по командам по дням.xlsx" --what-if "ONE-2515=1"
```

Программа пересчитает и покажет, **кого и на сколько дней это сдвинет**. Такой
прогон помечается как эксперимент и не влияет на обычные планы.

## 4. Записать план в Трекер (когда согласовали)

⚠️ Это **меняет даты в Трекере**. Делайте, только когда план проверен.
Безопасность встроена: перед записью программа сама сохраняет бэкап.

**Шаг 4.1 — сначала на одной задаче** (убедиться, что всё ок):
```powershell
uv run planner apply --plan "out\plan_ДАТА.json" --keys ONE-4711
```
Программа:
1. сделает свежий бэкап (`preapply_ДАТА.json`) и напечатает команду отката;
2. спросит подтверждение — наберите `yes` и Enter (или `нет` для отмены);
3. запишет и покажет результат.

Проверьте задачу ONE-4711 в Трекере. Всё верно?

**Шаг 4.2 — записать весь план:**
```powershell
uv run planner apply --plan "out\plan_ДАТА.json"
```
(подтверждение так же через `yes`).

> Подставляйте реальное имя файла вместо `plan_ДАТА.json` — его печатает шаг 1.
> Файлы лежат в папке `out\`.

---

## 5. Откатить, если что-то пошло не так

Вернуть даты, как были до записи:

```powershell
uv run planner restore --backup "out\preapply_ДАТА.json"
```
Имя `preapply_ДАТА.json` программа напечатала на шаге 4. Подтвердите `yes`.
Даты вернутся к состоянию до вашей записи (включая очистку тех, что были
пустыми).

Также можно снять снимок вручную в любой момент (например, перед экспериментами):
```powershell
uv run planner backup
```

## 6. Обновить исходные данные

- **Ресурс команд (отпуска, доступность).** В 1С в консоли запросов выполнить
  запрос `onec\Запрос_ГрафикДоступности.txt`, выгрузить результат в xlsx,
  положить рядом и указать его в `--capacity-file`. Подробности —
  в [onec/README.md](onec/README.md).
- **График релизов.** Файл `..\source\План релизов 1С.xlsx`. Сейчас заполнен
  до апреля 2026 — за этой датой программа берёт «релиз по средам» (помечает
  это в отчёте). Дозаполните, чтобы плановые даты были точными.

---

## 7. Что важно знать (ограничения текущей версии)

- **Тестировщики** пока не заведены как ресурс — после разработки закладывается
  фиксированные 5 рабочих дней на тест.
- **Бизнес-аналитика (БА)** в расчёт не входит (отдельная очередь).
- **Команда «ЗУП и ДО»** сейчас исключается: по ней в 1С не инициализирован
  ресурс. Как заведут — появится в плане автоматически.
- **Блокировки и связи** между задачами пока не учитываются.
- Задачи **без важности** и без оценки в план не попадают (см. лист
  «Пропущенные»).

## Шпаргалка команд

```powershell
cd "D:\Claude\yandex tracker analyzer\planner"                     # войти в папку

# посчитать план (ничего не меняет)
uv run planner plan --capacity file --capacity-file "..\source\ресурс по командам по дням.xlsx"

```

покажите текст ошибки тому, кто настраивал инструмент.
